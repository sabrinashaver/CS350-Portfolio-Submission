# CS350-Portfolio-Submission
**Project Summary:**
The artifacts included in this portfolio are my Module Four LCD Integration Lab and my Module Seven Smart Thermostat Final Project. Together, these projects demonstrate my progression from basic hardware setup to a fully integrated embedded system guided by technical and business requirements.
The Module Four projects focused on integrating a 16x2 display and potentiometer to expand system output beyond simple LED indicators. The final project built on this foundation by implementing a smart thermostat prototype that reads temperate data, manages operating states through a state machine, displays system information, and simulates data transmission to a server.

**Strengths:**
I successfully integrated multiple hardware components and ensured reliable communication between peripherals. In the final project, I implemented a clear state machine to control system behavior and correctly initialized GPIO, I2C, and UART peripherals. I also followed coding best practices through consistent formatting, commenting, and modular design.

**Areas for Improvement:**
I could improve system robustness by adding stronger error handling for unexpected sensor values and rapid user input. Further modularization and feature expansion would also enhance scalability.

**Tools, Skills, & Transferability:**
These projects strengthened my skills in hardware-software integration, state machine design, peripheral control, and system-level debugging. I also gained experience analyzing hardware architectures based on memory, peripheral support, and connectivity requirements, which is transferable to future coursework and embedded systems projects.

**Maintainability & Adaptability:**
Maintainability was achieved through clear naming conventions, modular functions, and in-code documentation. The use of a state machine allows the system to be easily extended or adapated for future features or hardware platforms.
